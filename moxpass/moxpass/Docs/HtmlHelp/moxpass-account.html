moxpass-account Manual Page

NAME

moxpass-account - Manage a moxpass account object.

SYNOPSIS

moxpass account [-h | --help] [list [-a | --all | <Friendly_Name> | <Login_Url>]] [add <Friendly_Name> [<Login_Url>] [<User_Identity>]] [<Friendly_Name> url-add <Login_Url>] [<Friendly_Name> url-del <Login_Url>] [<Friendly_Name> id-add <User_Identity>] [<Friendly_Name> id-del <User_Identity>]

DESCRIPTION

This command manages a moxpass account object under a specified moxpass login session.

There are certain operations that can only be done with an authenticated login session, while others do not.

OPTIONS

-h
--help

Prints the synopsis and a list of commonly used commands.

MOXPASS-ACCOUNT COMMANDS

moxpass account list

Print current moxpass account object details based on the arguments provided. Can print every object, or try to print a specific account object based on a friendly name, or a Login URL.

moxpass account add

Auth Required!

Create a new moxpass account object based on the arguments provided. Both login URL and user identity are optional.

It doesn't have to be an URL that works. If url doesn't work, moxpass will prompt the user asking if this is a mistake or intended. A common usage is to add a name of an application as the login URL.

moxpass account <Friendly_Name> url-add <Login_Url>

Auth Required!

Add a Login URL to a moxpass account object based on the friendly name supplied.

moxpass account <Friendly_Name> url-del <Login_Url>

Auth Required!

Delete a Login URL to a moxpass account object based on the friendly name supplied.

moxpass account <Friendly_Name> id-add <User_Identity>

Auth Required!

Add a user identity to a moxpass account object based on the friendly name supplied.

moxpass account <Friendly_Name> id-del <User_Identity>

Auth Required!

Delete a user identity to a moxpass account object based on the friendly name supplied.
