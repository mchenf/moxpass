moxpass Glossary

A

Account, moxpass account object

An object made up with one or multiple login URL with their respective user identity-secret pair.

It allows for an assignment of a friendly name to be consumed when invoking different moxpass commands.

Authenticated Session

An authenticated session is when user choose to sign-in for the defined period of time with moxpass app. During this period, user will not be prompt for the password repeatedly.

B

Block Code, ASCII Block Code

A 16 by 16 console graph made with ASCII block characters. Representing a SHA256 salted and hashed result of a given user secret.

The purpose of the ascii block code is to give user a familiarity of which secret is being used without actually rendering the literal password.

I

Identity-Secret Pair

An identity-secret pair is a combination of 1:1 relationship between a moxpass user identity and a moxpass user secret. If we are dealing with logins with sso (single sign-on), the same identity-secret pair can be reused.

L

Login URL, moxpass login URL

A moxpass login URL represents an endpoint where user is prompt to provide a user identity, i.e., email, phone, etc., so that an authentication can be made via a user secret. Usually, it is a namesake web url starting with http/https. Other formats are also accepted, i.e. descriptions like "MyNoteBookApp", virtual machines, database, or a local file url.

U

User Identity, ID, moxpass user identity

A moxpass user identity represents the part user is required to put in as "username" in any authentication prompt UI.

User Secret, moxpass user secret

A moxpass user secret usually represents the part user is required to put in as "password" in any authentication prompt UI. Note that a moxpass user secret cannot be created without an identity.
